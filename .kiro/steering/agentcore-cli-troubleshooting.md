# AgentCore CLI Troubleshooting

## CRITICAL: When `agentcore deploy -y` Fails with TypeScript/CDK Errors

If you see ANY of these errors:
- `'tsc' is not recognized`
- `Cannot find module '../lib/cdk-stack'`
- `Cannot find module 'aws-cdk-lib'`
- `Cannot find module '@aws/agentcore-cdk'`
- `moduleResolution=node10 is deprecated`
- Any TypeScript compilation error from `agentcore/cdk/`

### The Fix: Regenerate the ENTIRE `agentcore/cdk/` directory

Do NOT:
- Try to patch individual files
- Add `ignoreDeprecations` hacks
- Manually create `lib/cdk-stack.ts`
- Run `npm install` hoping it fixes things

DO:
1. Generate a fresh scaffold in a temp directory inside the workspace:
```powershell
agentcore create --project-name studentservices --name TempAgent --defaults --skip-git --skip-python-setup --skip-install --output-dir ".tmp-scaffold"
```

2. Replace the ENTIRE `agentcore/cdk/` directory:
```powershell
Remove-Item "workshop4/phase3/studentservices/agentcore/cdk" -Recurse -Force
Copy-Item ".tmp-scaffold/studentservices/agentcore/cdk" -Destination "workshop4/phase3/studentservices/agentcore/cdk" -Recurse -Force
```

3. Delete the temp scaffold:
```powershell
Remove-Item ".tmp-scaffold" -Recurse -Force
```

4. Install CDK dependencies:
```powershell
cd workshop4/phase3/studentservices/agentcore/cdk
npm install
```

5. Retry deploy:
```powershell
cd workshop4/phase3/studentservices
agentcore deploy -y
```

### What This Preserves (NEVER touched)
- All `agent.py` files (orchestrator + 4 specialists)
- `agentcore/agentcore.json` (runtime config, gateway, credentials)
- `agentcore/aws-targets.json` (deployment targets)
- `agentcore/.cli/` (deployed state, logs)
- `policies/` (Cedar files)
- `requirements.txt` files in each agent directory

### What This Replaces (disposable scaffold)
- `agentcore/cdk/bin/cdk.ts`
- `agentcore/cdk/lib/cdk-stack.ts`
- `agentcore/cdk/test/cdk.test.ts`
- `agentcore/cdk/package.json`
- `agentcore/cdk/tsconfig.json`
- `agentcore/cdk/node_modules/` (regenerated by npm install)

### Root Cause
The `agentcore` CLI bundles a TypeScript CDK app that synthesizes CloudFormation from `agentcore.json`. When the CLI is upgraded globally (`npm install -g @aws/agentcore`), it can regenerate parts of the scaffold or change expectations without running `npm install` locally. The scaffold is disposable infrastructure — the real config lives in `agentcore.json` and `aws-targets.json`.

### Key Flags for `agentcore create`
- `--skip-git` — DO NOT initialize a git repo (critical in existing repos)
- `--skip-python-setup` — DO NOT create venvs or install Python deps
- `--skip-install` — DO NOT run npm install (we do it manually after copying)
- `--output-dir` — Put it somewhere temporary, not in the real project
- `--defaults` — Skip interactive prompts (Python, Strands, Bedrock, no memory)
